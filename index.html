Network Layer

A VPN modifies routing logic at the operating system level. Traffic is redirected through a virtual interface such as wg0 or tun0.

Encapsulation

Original IP packets are encapsulated into encrypted transport frames. The ISP observes encrypted traffic toward a single endpoint.

Routing Control

The default gateway is replaced. Split tunneling modifies selective route tables.

DNS Resolution

If DNS is not routed internally, resolution leaks. Secure configuration ensures resolver binding to tunnel interface.

Encryption Layer

Modern VPN systems use authenticated encryption with ephemeral key exchange.

WireGuard

Implements the Noise IK handshake pattern. Curve25519 for key exchange. ChaCha20-Poly1305 for encryption and authentication.

Session Keys

Ephemeral keys rotate periodically. Perfect Forward Secrecy prevents retrospective decryption.

Cryptographic Discipline

Security depends on implementation integrity, not algorithm branding.

Infrastructure Layer

Infrastructure defines real trust boundaries. Encryption cannot override server compromise.

Jurisdiction

Legal frameworks may mandate metadata retention or disclosure.

Logging

“No logs” must exclude connection timestamps and IP mapping. Partial logging equals traceability.

Control Model

Self-hosted VPN increases control but increases operational responsibility. Commercial VPN centralizes risk.

Application Layer

VPN does not control application behavior.

WebRTC

Browsers may expose local IP through peer connection APIs.

Fingerprinting

Canvas entropy, GPU signatures and timing patterns form probabilistic identity.

Identity Layer

Authentication overrides transport masking.

Accounts

Logging into services binds session identity to user profile.

Behavior

Interaction patterns generate behavioral signatures independent of IP.

Conclusion

Encryption protects payload. Routing obscures destination. Infrastructure defines trust. Applications leak entropy. Identity persists above transport.

Security is layered discipline.
